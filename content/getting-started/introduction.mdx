# Introducción

> Conoce los conceptos básicos de la API de ESNT Ferreteria.

## ¿Qué es la API de ESNT Ferreteria?

La API de ESNT Ferreteria es un servicio REST que permite gestionar un sistema de ferretería completo. Proporciona endpoints para:

- **Autenticación**: Login, refresh token y logout
- **Empresas**: Gestión de múltiples empresas/ferreterías
- **Usuarios**: Creación y gestión de usuarios por empresa
- **Productos**: Catálogo de productos con categorías, marcas y proveedores
- **Stock**: Control de inventario por sucursal
- **Proformas**: Generación de proformas con PDF
- **Alertas**: Notificaciones de stock bajo

## Arquitectura

La API sigue una arquitectura por capas:

```
Route → Controller → Service → DAO → PostgreSQL
```

- **Route**: Define los endpoints HTTP
- **Controller**: Recibe y responde peticiones
- **Service**: Contiene la lógica de negocio
- **DAO**: Acceso a la base de datos

## Base URL

```
http://localhost:3000
```

## Formato de Respuesta

Todas las respuestas exitosas siguen el formato:

```json
{
  "campo": "valor"
}
```

Los errores siguen el formato:

```json
{
  "message": "Descripción del error",
  "statusCode": 400
}
```

## Autenticación

La mayoría de los endpoints requieren autenticación via JWT. Incluye el token en el header:

```
Authorization: Bearer <tu_access_token>
```

Consulta la sección de [Autenticación](/getting-started/authentication) para más detalles.

## Rate Limiting

La API aplica límites de velocidad:

| Tipo | Límite | Ventana |
|------|--------|---------|
| Auth endpoints | 10 requests | 1 minuto |
| Escritura (POST, PUT, PATCH, DELETE) | 60 requests | 1 minuto |
| Lectura (GET) | 120 requests | 1 minuto |

## Endpoints Públicos

- `GET /` - Health check
- `POST /auth/login` - Login
- `POST /auth/refresh` - Refresh token
- `POST /auth/logout` - Logout
- `GET /uploads/*` - Archivos estáticos
