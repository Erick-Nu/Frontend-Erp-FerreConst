# Autenticación

> Aprende a autenticar tus peticiones contra la API.

## Tipos de Token

La API utiliza dos tipos de tokens:

### Access Token

- **Duración**: 1 hora (configurable)
- **Uso**: Incluir en cada petición autenticada
- **Formato**: JWT

### Refresh Token

- **Duración**: 7 días (configurable)
- **Uso**: Renovar el access token cuando expira
- **Formato**: Token opaco (hexadecimal)

## Flujo de Autenticación

```
1. Login → Obtener access_token + refresh_token
2. Usar access_token en requests protegidos
3. Cuando expire, usar refresh_token para obtener nuevo access_token
4. Cuando quieras cerrar sesión, usar logout
```

## Login

### Request

```bash
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "emruc": "1709639106001",
    "usapodo": "jperez",
    "uspassword": "secreto123"
  }'
```

### Campos

| Campo | Tipo | Requerido | Descripción |
|-------|------|-----------|-------------|
| `emruc` | string | Sí | RUC de la empresa |
| `usapodo` | string | Sí | Apodo del usuario |
| `uspassword` | string | Sí | Contraseña del usuario |

## Refresh Session

### Request

```bash
curl -X POST http://localhost:3000/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{
    "refreshToken": "a1b2c3d4e5f6..."
  }'
```

### Respuesta

```json
{
  "accessToken": "nuevo_access_token...",
  "refreshToken": "nuevo_refresh_token..."
}
```

## Logout

### Request

```bash
curl -X POST http://localhost:3000/auth/logout \
  -H "Content-Type: application/json" \
  -d '{
    "refreshToken": "a1b2c3d4e5f6..."
  }'
```

## Seguridad

### Bloqueo por Intentos Fallidos

- Después de **3 intentos fallidos**, el login se bloquea por **15 minutos**
- El bloqueo aplica por combinación: `RUC + apodo + IP`
- El conteo incluye usuarios inexistentes (previene enumeración)
- Un login exitoso reinicia el contador

### Headers de Seguridad

La API aplica los siguientes headers via Helmet:

- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `X-XSS-Protection: 1; mode=block`
- `Strict-Transport-Security: max-age=31536000; includeSubDomains`

## Errores Comunes

| Código | Descripción |
|--------|-------------|
| 401 | Credenciales inválidas |
| 429 | Demasiados intentos fallidos |
| 500 | Error interno del servidor |
