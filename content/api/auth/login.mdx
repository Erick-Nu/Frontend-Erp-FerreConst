# Login

> Inicia sesión y obtén tokens de acceso para la API.

## OpenAPI

## Ejemplos

### Login básico

```bash
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "emruc": "1709639106001",
    "usapodo": "jperez",
    "uspassword": "secreto123"
  }'
```

### Login con curl y guardar token

```bash
TOKEN=$(curl -s -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "emruc": "1709639106001",
    "usapodo": "jperez",
    "uspassword": "secreto123"
  }' | jq -r '.accessToken')

echo $TOKEN
```

## Reglas de Negocio

- La empresa debe existir y estar en estado `activo`
- El usuario debe existir, pertenecer a la empresa y estar `activo`
- El RUC debe ser válido (13 dígitos)
- Después de 3 intentos fallidos, el acceso se bloquea por 15 minutos
- Un login exitoso reinicia el contador de intentos

## Errores

| Código | Mensaje | Descripción |
|--------|---------|-------------|
| 401 | Unauthorized | Credenciales inválidas |
| 429 | Too Many Requests | Bloqueo temporal por intentos fallidos |
| 500 | Company RUC is required | RUC no proporcionado |
| 500 | RUC must be valid | RUC con formato inválido |
| 500 | Nickname is required | Apodo no proporcionado |
| 500 | Password is required | Contraseña no proporcionada |
| 500 | User is inactive | Usuario en estado inactivo |
| 500 | Company is inactive | Empresa en estado inactivo |

## Notas de Seguridad

- El access token expira en 1 hora (configurable)
- El refresh token expira en 7 días (configurable)
- Los tokens se almacenan con hash SHA-256 en la base de datos
- El refresh token incluye IP y User-Agent para auditoría
