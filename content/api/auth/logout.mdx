# Logout

> Cierra la sesión revocando el refresh token.

## OpenAPI

```yaml /openapi.yaml post /auth/logout
openapi: 3.1.0
info:
  title: ESNT Backend Ferreteria API
  version: 1.0.0
paths:
  /auth/logout:
    post:
      summary: Cerrar sesión
      description: Revoca el refresh token para cerrar la sesión
      operationId: logoutUser
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/LogoutRequest'
            example:
              refreshToken: "a1b2c3d4e5f6..."
      responses:
        '200':
          description: Sesión cerrada exitosamente
        '401':
          description: Refresh token inválido
components:
  schemas:
    LogoutRequest:
      type: object
      required:
        - refreshToken
      properties:
        refreshToken:
          type: string
          description: Refresh token a revocar
```
```

## Ejemplos

### Cerrar sesión

```bash
curl -X POST http://localhost:3000/auth/logout \
  -H "Content-Type: application/json" \
  -d '{
    "refreshToken": "a1b2c3d4e5f6..."
  }'
```

### Logout completo (bash)

```bash
# Cerrar sesión y limpiar tokens
curl -X POST http://localhost:3000/auth/logout \
  -H "Content-Type: application/json" \
  -d "{\"refreshToken\": \"$REFRESH_TOKEN\"}"

# Limpiar variables
unset ACCESS_TOKEN
unset REFRESH_TOKEN
```

## Reglas de Negocio

- El refresh token se marca como revocado en la base de datos
- Un token revocado no puede usarse para renovar sesión
- El access token no se revoca (expira por tiempo)
- Se recomienda eliminar ambos tokens del cliente

## Errores

| Código | Mensaje | Descripción |
|--------|---------|-------------|
| 401 | Unauthorized | Refresh token inválido o ya revocado |

## Mejores Prácticas

1. **Siempre hacer logout** al cerrar sesión
2. **Eliminar tokens** del almacenamiento del cliente
3. **No reutilizar** refresh tokens después del logout
4. **Manejar errores** de red durante el logout
